Enterprise Scaling: Streamlit Web UI & Remote HR Operations

Led the architectural migration of standalone Python pipelines into a centralized, secure Streamlit web application, collaborating with developer assistance to deploy across a 7,000+ employee organization.

Streamlit Python Product Architecture UI/UX Secure File I/O Tailscale VPN

🎯 The Scaling Challenge (From CLI to Web App)

Following the success of my standalone Python ETL scripts, the organization required a way for non-technical HR specialists and regional managers to execute these data workflows autonomously. Running Python scripts from a command-line interface (CLI) and managing local file paths was not scalable across the department.

Operating as the Product Owner and Technical Architect, I authored a comprehensive Product Requirements Document (PRD V4.0) and collaborated with developer assistance to wrap the underlying Python engine in a secure, centralized web application.

🏗️ Architectural & Security Implementations

  • Centralized Streamlit Interface (`app.py`): Designed a dark-themed, intuitive UI featuring sidebar navigation for all 5 core HR workflows. Implemented real-time progress bars and spinners to maintain seamless user experience when processing 50-tab workbooks.
  • In-Memory Data Streaming (Zero PII Footprint): To adhere to rigorous data privacy policies, all file operations were re-architected to stream strictly in-memory using io.BytesIO. Raw biometric logs and payroll PII are processed entirely in RAM and immediately downloaded by the user, leaving zero footprint on the hosting server.
  • Secure Remote Access via Mesh VPN: Collaborated on configuring a hybrid deployment model using Tailscale encrypted mesh networking for secure remote execution directly against protected local data directories.

Python ETL & Data Wrangling Engine: ERP Attendance Pipelines

Engineered standalone Python CLI scripts from scratch to sanitize 50MB+ biometric exports, resolve Odoo database sync crashes, and parse wide-grid timesheets for 7,000+ employees.

Python Pandas Openpyxl Data Cleaning ETL Pipelines Odoo ERP

🎯 Project Overview & Core Mission

As the sole data specialist managing attendance and ERP integrations for over 7,000 construction employees across 50+ active sites, I encountered severe data infrastructure bottlenecks. Every month, raw biometric devices exported messy unformatted Excel logs, while departmental timekeepers submitted 30+ tab "wide-grid" timesheets that crashed Excel on standard laptops.

To eliminate hours of manual data entry and prevent recurring Odoo database transaction crashes during upload, I engineered a suite of standalone Python scripts to automate the entire data cleaning and transformation lifecycle.

💻 The Python Core Engine (Scripts Developed)

  • Biometric Attendance Cleaner: Automated the collapse of hundreds of raw, duplicate daily biometric punches per employee into exact shift start and end times using First-In / Last-Out (FILO) logic.
  • In-Place Database Injection: Developed an intelligent script that queries Odoo database exports, preserves hidden system record IDs, and injects sanitized hours directly into existing record shells—bypassing hardware sync crash loops entirely.
  • Wide-Grid Timesheet Extractor: Extracted monthly leave records from 25MB+, 50-tab Excel workbooks. To bypass memory limits, implemented openpyxl in read_only streaming mode. Consolidated consecutive daily leave codes into clean Start/End date ranges matching Odoo’s exact import schema.

📊 Individual Impact & Outcomes

  • 100% Autonomous Execution: Built and ran these scripts entirely via terminal CLI, reducing monthly data processing time from days to under 30 minutes.
  • Fuzzy Text Matching: Used Python's difflib and regex to standardize Arabic text variations, eliminating lookup failures.
